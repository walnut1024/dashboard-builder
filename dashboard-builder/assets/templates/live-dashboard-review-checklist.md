# Live Dashboard Review Checklist

- Screen optimizes for monitoring, not chart variety.
- Highest-priority metrics are stable in location and encoding.
- Last updated time and update cadence are visible.
- Live, delayed, stale, partial, offline, reconnecting, empty, and error states are distinct.
- Last-known-good evidence remains visible during reconnect or partial failure.
- Alerts have severity, source, time, and handling state.
- Filters and reset paths preserve orientation.
- Essential values are visible without hover.
- Performance degradation is defined for high update rates or dense marks.
- Export, incident snapshot, or screenshot path preserves the current state.
